Concurrent Git Flow Strategy

  • M1 Foundation First: base/boyang-foundation-identity-authentication — All Models, Services, Identity, and DB schema merged before any feature work.
  • Sequential PR Merges: feature/chunxi-business-crud (PR#1) → Feature/Marieth-Admin → feature/BoZhang-UI-Dashboard (PR#4) — each branch built atop the previous merge.
  • Hotfix Branch: Fix-accessDenied (PR#5) — post-merge patch adding AccessDenied page.

Project Setup

Foundation & Architecture

Set up the project structure including domain models, service layer with interfaces, Identity authentication, and global security filters.

  • Global Anti-Forgery: Every POST is automatically validated via AutoValidateAntiforgeryTokenAttribute in Program.cs.
  • 6 Scoped Services: Organization, Skill, Opportunity, Application, Coordinator, Volunteer — all injected via AddScoped.
  • Role Seeding: Admin, Coordinator, Volunteer roles auto-created at startup with a default admin account.

Identity & Security

Identity Configuration

  • ApplicationUser : IdentityUser<Guid> — Custom subclass with FullName and CreatedAt fields.
  • Lockout Policy: 3 failed attempts → 5 minute freeze, protecting against brute force attacks.
  • Cookie Paths: LoginPath = /Account/Login, AccessDeniedPath = /Home/AccessDenied configured in ConfigureApplicationCookie.
  • Auto Seed: 3 roles + admin account created on every app start via RoleSeeder.SeedAsync().

Authorization Design

Role-Based Access Control

ERD was covered in Part 1. In Part 2, we focused on how ApplicationUser connects to Identity roles and how each controller enforces access control.

  • Class-Level Auth: [Authorize] on the entire controller — all actions require login (e.g. Applications, Opportunities, Skills, Organizations).
  • Action-Level Override: Create/Edit/Delete actions use [Authorize(Roles = "Admin, Coordinator")] to restrict write operations.
  • Admin-Only: Coordinators, Organizations CUD locked to [Authorize(Roles = "Admin")].
  • Auto Registration: AccountController.Register auto-assigns "Volunteer" role via AddToRoleAsync after user creation.
Service Layer

Audit Tracking & Pagination

Each controller uses service interfaces via dependency injection. Audit fields track who created or modified each record.

  • Create Audit: CreatedByUserId = User.Identity?.Name stamps the creator on every new Application.
  • Edit Audit: LastModifiedBy + LastModifiedAt fields updated on every edit action.
  • DB-Level Pagination: GetPaginatedApplicationsAsync(pageNumber, 10) uses Skip().Take() at the EF query level.
// ApplicationsController.cs — Real audit tracking code [Authorize(Roles = "Admin, Coordinator")] [HttpPost] [ValidateAntiForgeryToken] public async Task<IActionResult> Create(Application application) { if (ModelState.IsValid) { application.CreatedByUserId = User.Identity?.Name; await _appService.AddApplicationAsync(application); return RedirectToAction(nameof(Index)); } // repopulate dropdowns on validation failure... }
// Edit action — tracks modification if (ModelState.IsValid) { application.LastModifiedBy = User.Identity?.Name; application.LastModifiedAt = DateTime.UtcNow; await _appService.UpdateApplicationAsync(application); }
Admin Panel

Admin User Management

Added AdminUsersController and AdminRolesController, both restricted to the Admin role with [Authorize(Roles = "Admin")].

  • User CRUD: Create users with password + role assignment, edit profile with role protection, and delete with safety checks.
  • Account Lock/Unlock: Admins can lock suspicious accounts using SetLockoutEndDateAsync, and unlock them later.
  • Self-Protection: Admin cannot lock, delete, or demote their own account.
// AdminUsersController.cs — Self-protection on Delete [HttpPost, ActionName("Delete")] [ValidateAntiForgeryToken] public async Task<IActionResult> DeleteConfirmed(Guid id) { var currentUser = await _userManager.GetUserAsync(User); if (currentUser != null && currentUser.Id == id) { TempData["Error"] = "You cannot delete your own account."; return RedirectToAction(nameof(Index)); } var user = await _userManager.FindByIdAsync(id.ToString()); if (user != null) await _userManager.DeleteAsync(user); return RedirectToAction(nameof(Index)); }
// Self-protection on Lock public async Task<IActionResult> Lock(Guid id) { var currentUser = await _userManager.GetUserAsync(User); if (currentUser != null && currentUser.Id == id) { TempData["Error"] = "You cannot lock your own account."; return RedirectToAction(nameof(Index)); } await _userManager.SetLockoutEndDateAsync(user, DateTimeOffset.UtcNow.AddYears(100)); }

Role Assignment Flow

AdminRolesController

Full CRUD on Identity Roles + role-user assignment with duplicate prevention and paginated display.

  • Assign GET: Builds AssignRoleViewModel with dual SelectList dropdowns + paginated user-role list from LINQ join.
  • Assign POST: Checks IsInRoleAsync before AddToRoleAsync to prevent duplicates.
  • RemoveAssignment: RemoveFromRoleAsync cleanly revokes a user's role.
  • SQL-Level Paging: GetAssignmentsQuery() uses LINQ join on UserRoles, Users, Roles tables with Skip/Take.
Dashboard

Dashboard Feature

Built the system dashboard with DashboardController, DashboardViewModel, and a Razor view showing live statistics and recent activities.

  • 6 Entity Counters: Volunteers, Organizations, Opportunities, Applications, Skills, Coordinators — each with emoji icon + hover animation.
  • Status Breakdown: Pending / Approved / Rejected progress bars calculated from TotalApplications.
  • Recent Activities: Top 5 Opportunities by EventDate with Organization name.
// DashboardController.cs — Full Index action [Authorize] public class DashboardController : Controller { private readonly AppDbContext _context; public async Task<IActionResult> Index() { var vm = new DashboardViewModel { TotalVolunteers = await _context.Volunteers.CountAsync(), TotalOrganizations = await _context.Organizations.CountAsync(), TotalOpportunities = await _context.Opportunities.CountAsync(), TotalApplications = await _context.Applications.CountAsync(), TotalSkills = await _context.Skills.CountAsync(), TotalCoordinators = await _context.Coordinators.CountAsync(), PendingApplications = await _context.Applications .CountAsync(a => a.Status == "Pending"), ApprovedApplications = await _context.Applications .CountAsync(a => a.Status == "Approved"), RejectedApplications = await _context.Applications .CountAsync(a => a.Status == "Rejected"), RecentActivities = await _context.Opportunities .OrderByDescending(o => o.EventDate) .Take(5) .Select(o => new RecentActivityItem { Title = o.Title, Description = o.Organization != null ? o.Organization.Name : "Unknown", Date = o.EventDate, Icon = "🎯" }).ToListAsync() }; return View(vm); } }
